# LifeQuest 2.0 (Desktop)

Курсовий/екзаменаційний проєкт. Це десктопна версія мого веб-застосунку LifeQuest — гейміфікований трекер завдань. Ти створюєш «квести» (свої справи), виконуєш їх і отримуєш за це XP, качаєш рівень, ранг і місце в таблиці лідерів.

Головна мета роботи — прикрутити до застосунку базу даних **SQL Server**. У веб-версії дані лежали в браузері, тут усе зберігається в БД.

## Що використано

- C# / WPF (.NET 10) — сам застосунок і інтерфейс
- SQL Server Express — база даних
- Microsoft.Data.SqlClient — для роботи з БД з коду
- Інтерфейс українською + є перемикання на English і світлу/темну тему

## Роль бази даних

У БД зберігається все, що потрібно, щоб застосунок працював між запусками:

- **Користувачі** — нікнейм, хеш пароля, мова, тема, прогрес.
- **Квести** — назва, опис, складність, статус, скільки XP нараховано.

Пароль у базі **не зберігається у відкритому вигляді** — тільки хеш (PBKDF2 + сіль). Усі запити до БД параметризовані, щоб не було SQL-ін'єкцій.

База і таблиці створюються **автоматично при першому запуску**, вручну нічого робити не треба (якщо бази `LifeQuest` немає — код її створює, потім створює таблиці `Users` і `Quests`).

### Таблиці

- `Users` — акаунт + прогрес (Xp, Lvl, TotalXp, Prestige, Multiplier, лічильники квестів, налаштування).
- `Quests` — квести користувача (Title, Descr, Difficulty, Status, HasProof, ProofPath, AwardedXp). Прив'язані до користувача через зовнішній ключ (FK), при видаленні користувача квести теж видаляються (ON DELETE CASCADE).

## Як працює гра

- XP за складність: Easy — 25, Medium — 50, Hard — 100.
- Якщо виконати квест без пруфу (фото), XP ділиться на 3.
- Формула для рівня: `100 × рівень^1.35`. Ранги міняються кожні 10 рівнів (Novice → ... → Legend).
- На день можна створити максимум 10 квестів.
- На 100 рівні можна зробити «престиж»: рівень скидається, але множник XP росте на +0.25 назавжди.
- Є таблиця лідерів за загальним XP.

## Адмін (аккаунт доступний тільки для розробника)

Користувач з нікнеймом **hlib** автоматично стає адміном. У нього є:

- Черга перевірки пруфів — коли гравець прикріпляє фото до квесту, воно йде на перевірку адміну. Адмін підтверджує → нараховується повний XP.
- Dev-режим — видати XP / рівень або скинути профіль (для тестів).

## Як запустити

Треба встановлений .NET 10 SDK і SQL Server Express (інстанс `.\SQLEXPRESS`).

```
dotnet run --project LifeQuest
```

Або відкрити `LifeQuest.slnx` у Visual Studio і натиснути запуск.

## Налаштування підключення до БД

За замовчуванням застосунок підключається до локального SQL Server:

```
Server=.\SQLEXPRESS;Database=LifeQuest;Trusted_Connection=True;TrustServerCertificate=True;Encrypt=False;
```

Рядок підключення можна змінити прямо в застосунку — Налаштування → База даних, там же є кнопка «Перевірити підключення». Він зберігається у файл `appsettings.json` поряд з .exe.

## Перевірка (тест)

У рішенні є окремий проєкт `SmokeTest` — він прогонює основну логіку через реальну БД: створення користувача, хешування пароля, створення квесту, чергу перевірки, нарахування XP, престиж, таблицю лідерів. Запуск:

```
dotnet run --project SmokeTest
```

Якщо все ок — виводить `ALL PASS`.

## Структура

```
LifeQuest-Desktop/
├── LifeQuest.slnx
├── LifeQuest/
│   ├── Data/         робота з БД (Database, репозиторії)
│   ├── Models/       моделі (User, Quest, Profile)
│   ├── Services/     ігрова логіка, хешування, локалізація, теми
│   ├── Themes/       світла/темна тема + стилі
│   ├── Views/        екрани (квести, статистика, лідери, налаштування, адмін, dev)
│   └── MainWindow    навігація + панель профілю
└── SmokeTest/        тест логіки через БД
```

## Що НЕ переносив з веб-версії

- Вхід через GitHub (OAuth) — для локальної програми немає сенсу, там потрібні веб-редіректи.
- Перевірку пруфів через ШІ — замінив на ручну перевірку адміном, щоб не тягнути зовнішній API-ключ, перевірка за допомогою ШІ буде доступна тільки після створення фінальної версії застосунка і публікації її на публіку.
